Mission: Develop Advanced Tenant Creation System
Context:
We are building an enterprise-grade multi-tenant Veterinary ERP SaaS on Replit. You need to develop a reusable TenantFormModal component for the System Admin panel (/system/tenants). This modal will be used for both creating new Tenants and editing existing ones.

🎯 Core Requirements:
1. User Interface (TenantFormModal):
Required Fields:

Tenant Code: Auto-generated (instant, non-sequential) - Format: VET-[6chars]-[timestamp]

Organization Name: Required text field

Subscription Plan: 4 options:

Trial: 7 days, max 3 users

Standard: 4 users

Professional: 6 users

Enterprise: Unlimited users

Country: Dropdown with Middle Eastern countries + Egypt only

Timezone: Changes based on selected country

Branding (Primary Color): Color picker

Contact Information: Email (validated), Phone, Address

Design:

Reusable Modal (used for both create and edit)

Two modes: create and edit

Buttons: Create/Update and Cancel

2. Backend System:
A. Unique Tenant Code Generation
Problem: 10+ System Admins may create tenants simultaneously

Solution: Lock-Free Code Generation system

Use Redis SETNX for uniqueness verification

Database fallback with retry logic

Code format: VET-A3B9C2-1733865234

B. Complete Tenant Creation Process
Single Service (TenantSetupService) that:

Saves basic tenant data

Sets user limits based on subscription plan

Copies DPF Structure from Template (not creating from scratch)

Adds heavy tasks to Queue (async)

C. DPF Template System
Massive performance improvement: From 26,180 DB operations per tenant → 1 operation

Create Template once on system startup

Copy Template for each new tenant

3. High Concurrency Handling:
Redis-based locking to prevent data conflicts

Expanded connection pooling in PostgreSQL

Queue system (BullMQ) for async tasks

Optimized database indexing

4. Subscription Management:
subscription_features table defining:

Allowed users per plan

Trial duration (7 days)

Features per plan

5. Scalability for 3,000+ Tenants:
PM2 Cluster Mode to utilize all CPU cores

Multi-level Redis Caching

Database Query Optimization with proper indexes

Monitoring with Prometheus Metrics

⚙️ Technical Requirements:
Integration with Existing System:
Path: /system/tenants

Add Tenant button: Opens TenantFormModal (mode='create')

Edit button: Opens same Modal (mode='edit') with current tenant data

A. Frontend Components:

B. Backend Services:

SubscriptionService.ts - Subscription plan management

C. Queue System:
tenantSetup.queue.ts - Background task processing

D. Database:
Migrations for subscription_features table

Performance-optimized indexes

Initial data for DPF Template

E. Monitoring:
tenantMetrics.ts - Prometheus metrics

Basic Grafana dashboard

F. Replit Configuration:
.replit - Run configurations

replit.nix - System dependencies

ecosystem.config.js - PM2 configuration
