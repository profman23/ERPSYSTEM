Bulletproof Concurrent Tenant Code Generation
Objective
Design and implement a 100% reliable tenant code generation system that guarantees zero duplicates and zero failures under high concurrent load (50+ simultaneous requests).

Critical Engineering Standards Required
1. Atomic & Lock-Free Architecture
No race conditions between SELECT check and INSERT operation

Database-level atomicity as primary guarantee (not just Redis)

Zero dependency on Redis for uniqueness (Redis optional for performance only)

2. Required Technical Implementation
Use PostgreSQL advisory locks OR SERIALIZABLE transaction isolation for true concurrency safety

Implement database sequence with INSERT ... ON CONFLICT DO UPDATE pattern

Fallback mechanism that works without Redis

Response time guarantee: < 100ms even under 50+ concurrent requests

3. Core Algorithm Requirements
text
1. Generate candidate code (VET-XXXXXX format)
2. Start SERIALIZABLE transaction
3. Attempt INSERT with ON CONFLICT handling
4. If conflict, generate new candidate and retry (max 3 attempts)
5. Commit and return success
