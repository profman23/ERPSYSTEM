# 1️⃣ SYSTEM REQUIREMENTS

## 🔥 Dynamic Permission Fabric (DPF)
Implement a system where:

- Every Module auto-registers itself.
- Every Screen auto-registers itself.
- Every Button/Action auto-registers itself.
- Every API endpoint auto-registers its required permissions.
- Every Event (socket.io) auto-registers its permission category.
- Future Modules register automatically (Zero-Touch RBAC).

DPF must store:
- Module ID  
- Screen/Submodule ID  
- Action ID  
- Action Category  
- Required AGI Level  
- Tenant Boundary  
- Scope (Tenant / Branch / Doctor / System)  
- Voice Command Mappings  
- Audit Trail  

All of this must be dynamically generated & self-maintained.
