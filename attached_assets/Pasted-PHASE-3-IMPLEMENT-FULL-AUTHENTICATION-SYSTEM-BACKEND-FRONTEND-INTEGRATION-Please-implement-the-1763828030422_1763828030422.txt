PHASE 3 — IMPLEMENT FULL AUTHENTICATION SYSTEM (BACKEND + FRONTEND INTEGRATION)
Please implement the complete enterprise authentication system for our Multi-Tenant Veterinary ERP SaaS platform using the exact instructions below.
Do NOT add any features beyond these requirements.

1) DATABASE SCHEMA UPDATES (DRIZZLE ORM)

Update the existing users table (DO NOT recreate it):

email (string, unique, indexed)

passwordHash (string)

role ("super_admin" | "tenant_admin" | "manager" | "staff")

isActive (boolean, default: true)

lastLoginAt (timestamp nullable)

Create a new table auth_tokens:

id (uuid)

userId (FK → users.id)

refreshToken (text)

expiresAt (timestamp)

createdAt (timestamp)

updatedAt (timestamp)

⚠️ IMPORTANT:
Do NOT modify tenants / business_lines / branches tables. They are already correct.

2) BACKEND AUTHENTICATION LOGIC
Password Hashing

Use bcryptjs:

hash password on creation

compare password on login

JWT Tokens

Use jsonwebtoken to generate:

Access Token (15 minutes)
Payload MUST include:

userId
role
accessScope
tenantId
businessLineId
branchId


Refresh Token (30 days)
Store in auth_tokens table.

3) AUTH ENDPOINTS
POST /auth/login

Request body:

tenantCode
email
password


Validation:

tenant exists

user exists

user belongs to tenant/businessLine/branch

password correct

user active

Response:

accessToken
refreshToken
user {
  id, name, email, role,
  accessScope, tenantId, businessLineId, branchId
}

POST /auth/refresh

Validate refresh token and issue new access token.

POST /auth/logout

Delete refresh token from DB.

4) MIDDLEWARE

Create:

authMiddleware

Validate JWT

Attach user to req.user

scopeGuard(requiredScope)

Block cross-tenant / cross-branch access

roleGuard(requiredRoles)

Allow routes only to specific roles

(Do NOT build granular permissions yet.)

5) FRONTEND INTEGRATION

Update LoginPage:

Send form to /auth/login

Show backend errors

Save accessToken + refreshToken in localStorage

Redirect to /dashboard

Handle token expiration by auto-refresh

Update ProtectedRoute:

Check accessToken

If expired: call /auth/refresh

If refresh fails: redirect /login

6) SEED SUPER ADMIN USER

Automatically create a system super admin during server startup:

email: superadmin@system.local
password: Admin@123
role: super_admin
accessScope: system
tenantId: null
businessLineId: null
branchId: null


Print to console:

Super Admin ready: superadmin@system.local / Admin@123

7) DO NOT IMPLEMENT ANYTHING ELSE

Do NOT implement the following in Phase 3:

Tenant CRUD

Business Line CRUD

Branch CRUD

User CRUD

Permissions system

Profile page

Forgot password

Change password

MFA

Any UI beyond LoginPage

Only Authentication.

✔ JWT validated by middleware
✔ ProtectedRoute fully active
